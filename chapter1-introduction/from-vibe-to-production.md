# Vibe Coding에서 프로덕션으로

## 문제: Vibe Coding

"프롬프트를 입력하고 기도하는" 방식의 코딩은 다음과 같은 문제가 있습니다:

- 코드가 의도에서 벗어남
- 문서화가 없고, 엣지 케이스가 고려되지 않음
- 프로토타입에는 좋지만, 프로덕션에서는 깨짐
- 유지보수가 어렵고, 팀원에게 인수인계하기 어려움

## 해결: Spec-Driven Development

스펙 기반 개발은 코드를 작성하기 전에 구조화된 계획을 수립합니다:

- 코드 작성 전에 **구조화된 계획** 수립
- **인수 기준**과 엣지 케이스가 포함된 요구사항
- 데이터 모델과 아키텍처를 포함한 **설계 문서**
- 하나씩 실행하는 **개별 태스크**

## 워크플로우

```
💬 아이디어 → 📋 요구사항 → 🏗️ 설계 → ✅ 태스크 → 🚀 코드
```

이 과정은 AI가 단순히 "코드를 생성하는 도구"에서 **"체계적으로 소프트웨어를 만드는 파트너"**로 전환하는 것을 의미합니다.
