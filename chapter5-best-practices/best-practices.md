# 베스트 프랙티스

> Kiro를 효과적으로 활용하기 위한 권장 사항과 주의 사항입니다.

## 권장 사항 (Do)

- **모든 기능은 Spec으로 시작하세요** — 구조화된 계획이 더 좋은 결과를 만듭니다
- **Steering 문서를 일찍 설정하세요** — 프로젝트 초기에 컨텍스트를 제공할수록 효과적입니다
- **학습 중에는 Supervised 모드를 사용하세요** — AI가 어떻게 코드를 생성하는지 이해할 수 있습니다
- **생성된 코드를 배포 전에 검토하세요** — AI는 파트너이지 대체물이 아닙니다
- **#으로 Kiro에 구체적인 컨텍스트를 제공하세요** — 더 정확한 결과를 얻을 수 있습니다
- **기술 스택에 맞는 Power를 설치하세요** — 관련 도구와의 통합이 워크플로우를 개선합니다

## 주의 사항 (Avoid)

- **복잡한 기능에 대해 스펙을 건너뛰지 마세요** — 계획 없는 코드는 문제를 만듭니다
- **"더 좋게 만들어줘" 같은 모호한 프롬프트를 피하세요** — 구체적인 요청이 더 좋은 결과를 만듭니다
- **생성된 요구사항을 무시하지 마세요** — 검토하고 수정하는 것이 중요합니다
- **코드 검토 없이 배포하지 마세요** — 항상 사람의 확인이 필요합니다
- **스티어링 문서에 비밀 정보를 넣지 마세요** — API 키, 비밀번호 등은 환경 변수로 관리하세요
- **하나의 스펙에 너무 많은 기능을 넣지 마세요** — 기능별로 분리하여 관리하세요

> **💡 참고**
**기억하세요**: Kiro는 당신의 파트너이지, 대체물이 아닙니다. 더 좋은 컨텍스트를 제공할수록 더 좋은 코드를 작성합니다.
